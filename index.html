Lesson 1: The Basics of Web and HTML

Important Information:


The World Wide Web:

The world wide web is made up of HTML files. These files are what makes the World Wide Web, "web-like." The Major pieces of the internet are, you(your computer), the internet and servers. HTTP is used to communicate between your computer and the web server.

HTML is made up of:

Text Content(what you see) Markup(what it looks like) References to other Documents(for example: images or videos) Links to other Pages


Tags:

HTML also uses tags. Tags are used so the computer can understand what you want it to do, or so you can tell the computer what you want something to look like.
In between those tags is the content.
The Tag and the Content together, make your element.


Imprtant things to remember:

When writing code, you need to be very precise with your imputs, because if your aren't, the computer will not understand what you are trying to tell it. In a sense, computers are "stupid."
You can't remember everything, so don't feel bad about having to look things up on Google(or whatever search engine is your preference).
There are also two different kinds of elements, block and inline.
Block organizes text into blocks and inline, well.....doesn't.


Lesson 2: Creating a Structured Document With HTML



Tree-like Structure

HTML uses a "tree-like" structure in a sense because you can have elements inside of elements and so on. This stucture shows up as a series of nested boxes. This structure makes it easier to read the code for the document.

Indented Boxes

When you read an HTML document as text you will see waves of indentations going up and down the document. When the content is more and more indented, it shows that it is more deeply nested into it's box.

Text Editors

Programmers use a text editor to write their code. Without these programs, writing code would be much more difficult, or even impossible. An example of this would be Sublime Text.


Lesson 3: Adding CSS Style and HTML Structure


What is CSS?

CSS is code that is written to control the "style" of HTML elements.
CSS stands for "Cascading Style Sheets."

How do divs work?

Div elemements are block elements used for containing blocks of content, which could be images or paragraphs of text for example.
They can also be given a unique id tag that can be linked to a CSS syle sheet.

Wait.....what does "Cascading" mean?

If you remember what we talked about with the "tree-like" structure of HTML, CSS uses that inheritance sturcture.
So in a sense, it cascades, passing on and on the same CSS style unless told otherwise.

The Box Model

HTML Boxes have 4 elements:
Margin: The Margin clears an are outside the border and is transparent.
Border: The border goes around the padding and the content.
Padding: The Padding clears an area around the content and is also transparent.
Content: The content is where the text and images appear. This is the stuff you can actually see.

Avoid Repitition:

Unnecesary repitition is only going to seem tedious, and is really unnecesary. When coding try to avoid writing the same things over and over again unless you have to for a specific reason.
